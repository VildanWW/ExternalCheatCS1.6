Всем привет! Хочу поделиться результатами своего первого проекта — External чита для Counter-Strike 1.6. На разработку ушел месяц, за который я на практике разобрался с архитектурой движка GoldSrc и манипуляцией памятью.

Почему именно External?
Главный плюс внешнего софта — работа в отдельном процессе. Мы не инжектим DLL напрямую, а взаимодействуем с памятью через системные вызовы. Это дает больше гибкости на старте и позволяет лучше понять, как клиент игры обрабатывает данные перед отправкой на сервер.

Функционал и технические детали:

ESP (Box/Visuals):
Реализован через чтение списка сущностей (Entity List). Чит считывает мировые координаты игроков и пересчитывает их в экранные с помощью матрицы (WorldToScreen). 
Нюанс: Я рассматривал возможность добавления Sound ESP (отслеживание по звуку), но из-за ограничений External-подхода и специфики задержек чтения памяти решил сфокусироваться на стабильной отрисовке боксов.<img width="982" height="570" alt="Снимок экрана 2026-02-04 195947" src="https://github.com/user-attachments/assets/1d8bf543-e503-403d-922d-f00fe12f19b3" />


Aimbot (Mathematical):
Наведение работает на основе векторных расчетов. 
Проблема: На данный момент аим испытывает трудности при стрельбе по целям, движущимся по трем осям одновременно. Это связано с отсутствием полноценного Bone Matrix (массива костей) в моей реализации. Сейчас наведение идет на центр модели с небольшим смещением (offset), что является временным решением. В будущем Internal проекте планирую реализовать прямой доступ к костям.

Bhop (BunnyHop):
Классическая реализация через проверку флагов игрока (FL_ONGROUND). Работает стабильно, хотя всегда есть пространство для оптимизации таймингов прыжка.

No Recoil (Control System):
Система компенсации отдачи. На скриншотах ниже видна разница в кучности стрельбы с включенной функцией и без неё.<img width="1204" height="647" alt="Снимок экрана 2026-02-04 192625" src="https://github.com/user-attachments/assets/21e54d4f-96f7-4586-a772-13d0c13d62d2" />

Итог:
Этот проект стал для меня отличным полигоном. Я столкнулся с ограничениями External-методов (задержки RPM, сложность поиска хитбоксов) и теперь готов двигаться дальше — в сторону Internal разработки, где доступ к игровым функциям и структурам позволит сделать чит на порядок быстрее и точнее.
